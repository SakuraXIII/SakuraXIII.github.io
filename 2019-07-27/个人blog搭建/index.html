基于Hexo框架的个人blog搭建

前言：

之前朋友搞个服务器，用了Wordpress环境，想要搭建个人博客，让我来帮忙给页面美化美化，但是Wordpress是基于php环境运行的，碰巧本人刚好不会，于是从各个地方找到Wordpress的搭建教程，断断续续搞了很久，也没有搞出什么名堂。

最近开始尝试着在CSDN上发表的两篇博客，感觉挺有意思的，越发喜欢使用Markdown写文章，越发想要创建自己的个人博客。

终于，昨天在无意间听说了Github Pages这个名词，于是去百度查了一下，又去GitHub官网去看了一下官方的介绍（如下）：

GitHub Pages are public webpages freely hosted and easily published through our site。

GitHub Pages可以被认为是用户编写的、托管在github上的静态网页，是个免费的host服务。

这个功能可以用来创建属于自己的静态页面，提供了1G的存储空间，用来搭建个人博客再好不过了。

了解Github Pages的同时，又在简书、CSDN各大论坛了解到了Hexo、Jekyll，这两家都是体供静态博客的主流框架，可以配合Github Pages快速搭建属于自己的个人博客，并且他们都是解析Markdown编写的文章，需要有Markdown的语法知识。

简单了解一下两大框架的区别，Jekyll是基于Ruby打造的，Hexo是基于Node.js打造的，身为一个半吊子的前端萌新，自然是更喜欢偏向与前端的Node.js，所以我选择的Hexo（然而，在GitHub上创建了GitHub Pages后在仓库的“设置(Settings)”中的“选项(option)”中直接提供了Jekyll的主题选择选项QAQ）。

所以，这篇文章将会是属于Hexo的教程。

创建Github Pages

在一切的开始之前，需要拥有两样东西：Node.js、git。

前期准备工作完毕后就可以正式开始了，首先肯定是要有一个GitHub的账户的，登陆个人账户后，创建一个新的repository，存储库的名称需要符合特定的规范：[username].github.io ，username即为自己的GitHub用户名，这样可以保证每个人的博客域名不会出现冲突。命名完成后，初始化一下README.md就可以创建啦。

创建完成以后，随便写一个demo，测试一下是否可以用仓库名访问到内容。如果没有问题，就可以开始下一步了。

文件夹下面有一个_config.yml 的文件，这里面都是相关的配置信息，初次下载后，需要在里面配置一下个人信息，使用文本编辑器（建议不要使用记事本这种上古神器）打开后，可以修改里面的一些信息：

Site

  • title 网站标题
  • subtitle 副标题
  • description 网站描述
  • keyword 关键字 用于SEO
  • author 作者
  • language 语言 国人就用zh-CN吧，不设置也可以
  • timezone 时区 默认使用个人电脑的时区

以上这些东西按照个人的需求配置即可。

  • url 配置为自己的个人站点
  • theme 为个人博客的主题样式，下载了新的主题后更改这里的名字，即可生效
  • deploy 下的 type 为 git ，repo 为自己的GitHub Pages仓库的地址，在添加一个选项 branch，为仓库的分支，如果是默认的master，也可不写
